# WordPress Migration Plan - Orders & Customers Only

## OVERVIEW
Migrate orders and customers from remote WordPress (37.27.192.145) to local setup.

## DATA ANALYSIS
- **Remote DB**: nilgiristores_in_db (prefix: kdf_)
- **Orders**: 2,375 total (711 delivered, 948 failed, 506 completed, etc.)
- **Users**: 29,272 total
- **Existing Tools**: Working scripts in wp_to_html/ folder

## MIGRATION PHASES

### Phase 1: Setup (2h)
- Configure migration settings
- Test database connections  
- Backup local database

### Phase 2: Customers (4h)
- Migrate: kdf_users → wp_users
- Migrate: kdf_usermeta → wp_usermeta
- Filter customer accounts only
- Preserve passwords/metadata

### Phase 3: Orders (6h)
- Migrate: kdf_posts (shop_order) → wp_posts
- Migrate: kdf_postmeta → wp_postmeta  
- Migrate: kdf_woocommerce_order_* tables
- Preserve order-customer relationships

### Phase 4: Validation (2h)
- Verify counts match
- Test WooCommerce functionality
- Validate customer login/order history

**Total Time: 14 hours**

## SCRIPTS TO CREATE
1. `migrate_customers.sh` - Customer migration
2. `migrate_orders.sh` - Order migration  
3. `validate_migration.sh` - Data validation
4. `rollback.sh` - Emergency rollback
5. `main_migrate.sh` - Master script

## RISKS & MITIGATION
- **Data Loss**: Full backup before start
- **ID Conflicts**: Careful user ID mapping
- **Order Integrity**: Maintain relationships
- **Rollback**: Emergency recovery scripts

## SUCCESS CRITERIA
- All 2,375 orders migrated
- Customer accounts functional
- Order-customer relationships preserved
- WooCommerce checkout working
